## Generate frourio project {#generate-frourio-project}

  defaultValue="npx"

<TabItem value="npx">

```sh title='Shell'
npx create-frourio-app --answers '{"dir":"my-project"}'
```

```sh title='Command Prompt'
npx create-frourio-app --answers ^"{\^"dir\^":\^"my-project\^"}^"
```

</TabItem>
<TabItem value="npm">

```sh title='Shell'
npm init frourio-app --answers '{"dir":"my-project"}'
```

```sh title='Command Prompt'
npm init frourio-app --answers ^"{\^"dir\^":\^"my-project\^"}^"
```

</TabItem>
<TabItem value="yarn">

```sh title='Shell'
yarn create frourio-app --answers '{"dir":"my-project"}'
```

```sh title='Command Prompt'
yarn create frourio-app --answers ^"{\^"dir\^":\^"my-project\^"}^"
```

</TabItem>
</Tabs>

With the `--answers` option, create-frourio-app starts in CUI mode.Refer to [the internal code](https://github.com/frouriojs/create-frourio-app/blob/main/server/common/prompts.ts) for json specifications as needed.

When generation is complete, [localhost:8000](http://localhost:8000) will be opened in your browser and the client framework you selected will be displayed.
(If it does not open automatically, open [localhost:8000](http://localhost:8000) manually.)

:::caution
If another application is using port 8000, it will probably fail to start.
In that case, the generation is already completed, so quit the application and start frourio again manually, e.g. `npm run dev`.
:::
